# <a name="manage-floor-plans"></a>フロアプランを管理する

Microsoft の検索フロアプランは、ユーザーが建物内の人や会議室を見つけやすくするために役立ちます。 フロアプラン次の質問に回答してください。
- Allan の office はどこにありますか?
- 会議室 C1 はどこにありますか?

![フロアプランマップ建物内でのユーザーのオフィスの場所を指定します。](media/floorplans-officelocation.png)

このような質問に対する回答を簡単に見つけられるように、組織の建物、オフィス、施設に関する情報を入手して、検索可能にする必要があります。 大規模な組織では、通常、設備またはスペースの管理チームが存在し、この情報が既に提供されている可能性があります。 小規模な組織では、検索管理者がそれを作成して追加する必要がある場合があります。

## <a name="48-hours-before-you-begin"></a>開始する前に48時間
フロアプランのアップロードを開始する前に、ユーザーのオフィスの場所にインデックスを作成する必要があります。 組織の規模によっては、完了するまでに最大48時間かかる場合があります。 この手順を無視した場合は、手順を実行しているときにエラーが表示されます。

![「フロアプランをアップロードする前に、Microsoft は office の場所を収集して整理する必要がある」という [フロアプランの画面キャプチャ] ページ](media/floorplans_hydrationstep.png)

Microsoft 365[管理センター](https://admin.microsoft.com)で、[**設定** > ]**microsoft 検索** > **フロアプラン**に移動し、[**開始**] を選択します。

この通知が表示されない場合は、組織内のユーザーまたは組織内の誰かがこの手順を既に開始しています。

## <a name="things-to-consider"></a>考慮事項
ユーザーがオフィスや施設に関する情報を見つけられるようにするには、次のものを追加する必要があります。

|十分     |これが重要な理由  |
|---------|---------|
|建物の場所    |    各建物を Microsoft の検索場所に追加する必要があります。 各建物の標準の名前付け形式が表示されます。 ストリートアドレスまたはマップ座標を使用して、建物を追加できます。     |
|すべてのユーザーアカウントの**Office**プロパティ     |    各ユーザーアカウントには、office の場所に**office**のプロパティを設定する必要があります。 およびオフィスの場所は標準形式に従う必要があり、建物、フロア、および会議室の情報が含まれます。   <br> Azure AD では、このプロパティを**Physicaldeliveryofficename**と呼びます。    |
|DWG 形式のフロアプランファイル     |   建物の床や主翼ごとに個別のフロアプランを必要とし、ユーザーの Office プロパティで使用したのと同じ形式で office 情報を含める必要があります。 ファイルは、AutoCAD 図面 DWG 形式である必要があります。 |

詳細については、「 [Microsoft 検索フロアプランのベストプラクティス](floorplans-bestpractices.md)」を参照してください。

## <a name="building-location"></a>建物の場所

場所として追加する必要がある建物を特定します。 建物の位置のアドレスとマップの座標は、最初の識別ポイントになります。 建物がまだ場所として追加されていない場合は、管理者がそれを追加する必要があります。 詳細については、「[場所の管理](manage-locations.md)」を参照してください。

## <a name="floor-plans-files"></a>フロアプランファイル

建物を識別した後、フロアプランを追加できます。 すべてのフロアプランファイルは、DWG 形式である必要があります。 組織にまだ設定されていない場合は、DWG 互換のアプリでフロアプランを作成する必要があります。 フロアプランは、会議または会議室、restrooms、キッチン、メールルーム、その他の施設を含むすべてのルームを適切にマッピングして、検索を有効にする必要があります。

### <a name="office-locations"></a>オフィスの場所

フロアプランにマップするには、すべてのオフィスの場所と従業員の office データがユーザーのアカウントにある必要があります。 フロアプランでは、オフィスの場所は一意である必要があり、繰り返すことはできません。 たとえば、2人のユーザーが office 2/1173 を共有している場合、 **2/1173**はフロアプラン内に1つの一意のインスタンスしか持つことができますが、Azure AD のユーザーアカウントは両方とも同じオフィスの場所を持つことになります。

![floorplans-peoplecard](media/floorplans-peoplecard.png)

 > [!Note] 
 > ユーザーが仕事仲間の会議室またはオフィスの場所を検索すると、フロアプラン内の部屋番号は Azure AD のオフィスの場所と一致します。 一致するものが見つかった場合は、マップが表示されます。

## <a name="add-floor-plan"></a>フロアプランを追加する

 フロアプランに初めて移動すると、ページの上部に、*フロアプランをアップロードする前に office の場所を収集して整理する必要*があるというメモが表示されることがあります。 [**開始**] を選択して、Azure AD office の場所にインデックスを作成します。 

1. [管理センター](https://admin.microsoft.com)で、[**設定** > ] の [**Microsoft 検索** >**フロアプラン**] に移動し、[**フロアプランの追加**] を選択します。
4. ドロップダウンで建物を選択し、[**次へ**] を選択します。 建物が表示されていない場合は、場所に追加する必要があります。 詳細については、「[場所を管理](manage-locations.md)する」を参照してください。
6. [**ファイルのアップロード**] を選択し、アップロードするフロアプランを選択します。 
1. ファイルを正常にアップロードした後、フロア番号または翼がどのように表示されるかを特定する必要があります。 
7. 建物を識別するコードを入力します。 ビルディングコードは、 **Office location**プロパティのユーザーのアカウントにあります。 たとえば、ユーザーのオフィスの場所が**2/1173**の場合、コードの作成は**2**です。 
9. アップロードされたすべてのフロアプランの場所パターンを確認して特定し、[**次へ**] を選択します。
10. 準備ができたら、[**発行**] を選択して、検索可能なフロアプランを作成します。

> [!Note] 
> フロアプランが下書き状態のときは、不完全です。 原案を使用すると、フロアプランのアップロードと作成に関係者を調整できます。 また、フロアプランを段階的に展開することもできます。

## <a name="edit-floor-plans"></a>フロアプランを編集する

1. [管理センター](https://admin.microsoft.com)で、[**設定** > ] の [**Microsoft 検索** > **フロアプラン**] に移動します。 
1. [**発行済み**] または [**下書き**] を選択して、変更するフロアプランを選択し、[**編集**] を選択します。
5. 変更を行い、[**保存**] を選択します。

## <a name="troubleshoot-errors"></a>エラーのトラブルシューティング

すべてのエラーが修正されるまで、フロア、翼、および部屋情報を定義する次の手順に進むことはできません。 次の表ファイルは、問題を解決するためのエラーメッセージとアクションをアップロードします。

| エラー メッセージ   | 種類    | Action       |
|:----------------| :--------- | :-------------- |
| CC_1 を読み取ることができません。 フロアプランを再アップロードまたは削除してください。 | Error |  ファイルをもう一度アップロードしてください。 それでも問題が解決しない場合は、ファイルを削除して、もう一度実行してください。 |
| CC_1 という名前のファイルが2つあります。 いずれかを削除するか、別の名前で再アップロードしてください。| Error | ファイル名が正しくない場合は、floor またはウィング情報を追加してファイル名を一意にしてから、もう一度ファイルをアップロードしてください。 <br><br>誤って同じファイルを追加した場合は、そのファイルを削除するだけです。 |
| データが見つかりません。 | Error | ファイルが正しいことを確認してから、もう一度アップロードするか、削除してください。 |
| このファイルに外部参照が含まれていません。 "CC_1_furniture" をアップロードするか、またはこのファイルを削除します。 | 警告 | 外部参照ファイルをアップロードするか、削除します。|
| DWG ファイルの部屋番号またはタグを読み取ることができませんでした。 このファイルを削除してください。 | 警告 | DWG ファイルをチェックしてデータが含まれていることを確認してから、ファイルを削除して、もう一度実行してください。 |
